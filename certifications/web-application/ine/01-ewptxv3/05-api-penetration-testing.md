# API Penetration Testing - eWPTXv3

## Overview
Comprehensive API penetration testing including REST, GraphQL, and SOAP APIs.

**Source**: [dev-angelist GitBook](https://dev-angelist.gitbook.io/ewptxv3-notes/api-penetration-testing)

## Learning Objectives
- Conduct hands-on penetration tests on API endpoints
- Utilize automation tools for API vulnerability testing
- Analyze API endpoints for parameter manipulation vulnerabilities
- Identify vulnerabilities related to rate limiting
- Demonstrate ability to bypass or manipulate rate limiting mechanisms

## API Testing

### API Types
- REST APIs
- GraphQL APIs
- SOAP APIs
- gRPC APIs

### API Discovery
- Endpoint enumeration
- Documentation analysis

- Authentication mechanisms

### Testing Techniques

#### Parameter Manipulation
- Query parameters
- Path parameters

- Header manipulation

#### Rate Limiting
- Testing rate limits
- Bypass techniques
- DoS potential
- Resource exhaustion

### Common Vulnerabilities
- Broken authentication
- Excessive data exposure
- Lack of rate limiting
- Mass assignment
- Injection vulnerabilities
- Insecure direct object references
